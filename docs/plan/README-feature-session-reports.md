# Feature: Session and Report Workflow

## Goal
Let a designer save a work session and export a clear record of why a blueprint was chosen.

## What It Should Show
- Saved sessions
- Notes and rationale
- Exportable review reports

## Why It Matters
The dashboard should support decision-making over time, not only one-off browsing.

## Suggested UI
- Save session action
- Notes panel
- Report export

## UI Shape
- A session sidebar or modal
- Timestamped entries with saved filters, selected blueprints, and notes
- Export to Markdown or JSON for design review

## Data Needed
- Session state snapshots
- Note storage
- Report serialization

## Status
Not implemented.
