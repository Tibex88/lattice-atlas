# Feature: Design-Language Balance Reports

## Goal
Translate raw lattice metrics into dashboard language that a designer can use directly.

## What It Should Show
- Variety pressure from width
- Progression depth from height
- Redundancy risk
- Over-centralization risk

## Why It Matters
The math is already visible, but the workbench still needs to explain what those shapes imply for player choice and content structure.

## Suggested UI
- Summary badges
- Risk callouts
- Short textual interpretations beside charts

## UI Shape
- A balance-report panel attached to the selected blueprint
- Metric cards paired with short interpretations
- Drill-down links that highlight the entries or branches driving the warning

## Data Needed
- Width and height metrics
- Property and expansion summaries
- Derived heuristics for design-facing labels

## Status
Not implemented.
