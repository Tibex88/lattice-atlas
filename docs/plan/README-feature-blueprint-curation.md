# Feature: Blueprint Curation

## Goal
Let the dashboard save, tag, and compare candidate algebraic shapes before any content mapping begins.

## What It Should Show
- Saved candidate blueprints
- Tags such as `movement`, `combat`, or `spell logic`
- Shortlist comparison views

## Why It Matters
Shape-first design needs a stable place to collect and compare promising structures instead of rediscovering them through raw browsing.

## Suggested UI
- Save or favorite actions on entries
- A saved-blueprints panel
- Tag and note fields per saved blueprint

## UI Shape
- A persistent shortlist drawer or page
- Compact blueprint cards with width, height, key properties, and expansion counts
- Quick compare actions to load shortlisted shapes into the existing analysis views

## Data Needed
- Local saved blueprint metadata
- References to dataset, `n`, and entry index

## Status
Not implemented.
